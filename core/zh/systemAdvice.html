Settings for on (local)

Java

The interactive diagrams in CAST are Java programs. Since Java cannot be used on Android or Apple tablets, the interactive versions of CAST pages are therefore not available to you.

Although you will be able to see the pages in this e-book, their interactive diagrams will be blank. Since CAST has been structured to explain statistical concepts using its interactive diagrams, you will not find it easy to read.

Since this e-book has video or summary versions of its pages, you will be able to read them, but the interactive versions of the pages will not be available.

Since the interactive versions of the CAST pages are such an important part of the e-book, we recommend that it is studied on a full computer (laptop or desktop) rather than a tablet or smartphone.

The interactive material in CAST relies on Java, so it is important to install and enable it.

If a green rectangle appears without warnings above, Java is working fine. (It may take a minute or two to appear.)
If it does not appear, first ensure that you have the latest version of Java installed.

Even after Java has been installed, you may need to change settings in your operating system or browser in order to fully enable it (and see the green rectangle above without warnings).

Enabling Java 7

If you are using the latest version of Java (Java 7), it must be enabled in the Windows Control Panel. In the Windows 8 desktop, you can access the Control Panel by either right-clicking the bottom left of the screen or hold down the Windows key and press 'X'.In Windows 7 or earlier, the Control Panel is accessed from the Start menu.

When the Control Panel is visible, select the Java Control Panel, then click the Security tab. In the Advanced tab, reduce the security level from High to LowMedium.

Internet Explorer Settings

Internet Explorer 8 and earlier do not support web standards for video, so the videos that are present in this e-book cannot be watched. To see the videos, we recommend either using Firefox or Chrome of upgrading to Internet Explorer 9. The original interactive material in CAST still works.

Click the Tools button then click Internet Options.

  • In the Advanced tab, select the option Allow active content from files on this computer.
  • In the Security tab, turn off the option Enable Protected Mode.

Firefox Settings

When using a locally-installed copy of CAST, Firefox always shows a warning instead of each interactive diagram (Java applet). As a result, every diagram must be clicked to dismiss this warning before it will be displayed.

There is currently no way to tell Firefox to globally enable Java when using a local copy of CAST, so we recommend using a different browser.

By default, Firefox always shows a warning instead of each interactive diagram (Java applet) in CAST. Individual diagrams can be clicked to dismiss these warnings.

These warnings can be dismissed globally for CAST. When a page with an applet is displayed, a red 'Lego block' is shown next to the URL at the top left of the window. Click this to display a message window about the Java plugin. Click the Activate All Plugins dropdown menu and choose Always activate plugins for this site.

More information is available here.

Chrome Settings

When Chrome first shows a CAST page with an interactive diagram, it asks at the top of the window for permission to run Java.

If you click Always run on this site, Chrome display the interactive diagrams in CAST for the rest of this session. Since this is a locally installed copy of CAST however, Chrome will ask again when CAST is restarted later.

Chrome imposes extra "security" restrictions when running a local copy of CAST and these prevent the index and list of data sets from communicating with the main CAST window. If you want to use these features in CAST, we recommend using Internet Explorer or Firefox.

If you click Always run on this site, Chrome display the interactive diagrams in CAST from then on.

Enabling Java 7

If you are using the latest version of Java (Java 7), it must be enabled in the Mac System Preferences. This can usually be found on the Dock.

When the Preferences window is visible, select the Java Preferences, then click the Security tab. In the Advanced tab, reduce the security level from High to LowMedium.

Safari Settings

When running CAST from a locally-installed copy, a few changes must be made to Safari's settings before it will display CAST's interactive diagrams (Java applets).

  • Select Preferences from Safari's File menu.
  • In the Advanced settings, select Show Develop menu in menu bar. Safari will now add an extra menu called Develop to the menu bar.
  • In the Develop menu, the option Disable Local File Restrictions must be set for the interactive diagrams to appear.

Safari on Lion or a more recent version of MacOS X imposes extra "security" restrictions when running a local copy of CAST and these prevent the index and list of data sets from communicating with the main CAST window. If you want to use these features in CAST, we recommend using Firefox.

By default, Safari displays a pop-up warning when you look at any CAST page containing an interactive diagram, asking whether you want to Allow or Block it.

  • Click Allow.
  • Select Preferences from Safari's File menu.
  • In the Security settings, make sure that Allow Java is ticked, then click Manage Website Settings.
  • In the dialog box that appears, there is a blank entry that represents locally installed files. Change its settings to Allow Always.
  • In the dialog box that appears, there is an entry corresponding to the CAST web site. Change its settings to Allow Always.
