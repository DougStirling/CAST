Modellierung anderer Situationen mit Wahrscheinlichkeit

Oft modelliert man einen Datensatz aus zufälligen Stichproben einer Grundgesamtheit und Wahrscheinlichkeit, die eingeführt wurde als eine Art der Beschreibung des Zufalls solcher Daten. Wahrscheinlichkeit wird auch für die Darstellung verschiedener anderer Situationen die Zufall beinhalten, benutzt.

Die Zufälligkeit von Glücksspielen, die Karten, Würfel oder Rouletteräder beinhalten, kann oft einfach in Wahrscheinlichkeiten ausgedrückt werden. Auch Sportwettkämpfe können oft dargestellt werden, indem man recht einfache Wahrscheinlichkeitsmodelle benutzt. Solche Darstellungen vereinfachen normalerweise die Realität, aber sie können den Kern des Verhaltens erfassen.

Frauen-Tennis-Match

Ein simples Modell für ein Tennis Match zwischen zwei Spielern, A und B, wird nun beschrieben. In diesem Modell nehmen wir folgendes an:

Wahrscheinlichkeit, dass A den Aufschlag gewinnt = π1
Wahrscheinlichkeit, dass B den Aufschlag gewinnt = π2

Wir gehen auch davon aus, dass die Ergebnisse von aufeinanderfolgenden Punkten unabhängig sind - wenn A einen Punkt gewinnt, heißt das nicht, dass sie eine höhere Chance hat, den nächsten Punkt zu gewinnen - und die Standardregeln des Tennis sind auch als Teil des Modells zu beachten. Dies ist eine Vereinfachung eine richtigen Tennis-Matches, aber es ist trotzdem hilfreich zu bestimmen, inwiefern die Wahrscheinlichkeit das gesamte Match zu gewinnen von π 1 und π 2 abhängt.

Simulation

Wie kann ein Wahrscheinlichkeitsmodell zur Informationssuche über ein solches System dienen? Zum einen können die Wahrscheinlichkeiten genutzt werden, um einen Fall des Systems zu erstellen. Wenn das Modell zu spezifisch war, dass etwas mit der Wahrscheinlichkeit von 0,5 passiert, dann könnte man eine Münze werfen, um einen Fall zu erstellen (mit z.B. Kopf = Eintreten des Ereignisses). Ereignisse mit anderen Wahrscheinlichkeiten können auf eine ähnliche Weise auf einem Computer erstellt werden.

Alle "Ereignisse" in einem Modell mit den Wahrscheinlichkeiten auf diese Art und Weise zu generieren, nennt man Simulation des Modells.
(Der Mechanismus wird im unteren Beispiel aufgeklärt).

Frauen-Tennis-Match

Das untere Diagramm zeigt wie zufällig generierte Punkte ein komplettes Frauen-Tennis-Match mit 3 Sätzen simulieren können. In der Anfangseinstellung sind beide Spieler ausgeglichen und haben eine Wahrscheinlichkeit von 0,75 ihren Aufschlag zu gewinnen.

Klick Simuliere nächsten Punkt, um einen einzelnen Punkt im Match zu spielen - der Computer generiert ein zufälliges Ergebnis, das auf der Wahrscheinlichkeit basiert, dass der Spieler mit dem Aufschlag gewinnt. Klicke wiederholt auf die Schaltfläche, um Punkte zu generieren bis das Match vorbei ist. (Oder halte den Schalter gedrückt, um die Simulation zu beschleunigen).

Klicke Starte neues Match, um noch eine Simulation durchzuführen. Beachte, dass es unwahrscheinlich ist, dass die genaue Abfolge der Punkte in anderen Simulationen genauso wiederholt wird, sogar wenn die Wahrscheinlichkeiten in aufeinanderfolgenden Matches dieselben sind. (Bevor man weitere Simulationen durchführt, kann man die Schieberegler benutzen, um die Wahrscheinlichkeiten so zu beinflussen, dass ein bestimmter Spieler Punkte erhält.)

In der Praxis wäre man kaum so interessiert, eine Simulation mit solch einer Genauigkeit darzustellen (außer vielleicht wenn man überprüfen möchte, dass man die Regeln des Matches richtig programmiert hat.) Normalerweise sind wir nur an ein oder zwei Ergebnissen interessiert (wie zum Beispiel die Identität des Gewinners oder die komplette Anzahl an Sätzen, die im Match gespielt wurden) und nur diese Zusammenfassungen jedes Durchlaufs der Simulation sollen dargestellt werden.
