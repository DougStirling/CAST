Rechenbetontes Problem

Um einen p-Wert für einen Hypothesen-Test eines Verhältnisses finden zu können, müssen die Restwahrscheinlichkeiten für eine Binomialverteilung summiert werden.

Wenn die Stichprobengröße n groß ist, kann es eine riesige Anzahl von Wahrscheinlichkeiten geben, die man zusammenfassen muss. Das ist nicht nur ermüdend, sondern kann auch zu einer Reihe von numerischen Fehlern führen.

Schädliche Reaktion auf Medikamente

Ein Pharmazie-Unternehmen behauptet, dass nur 1% der Verbraucher eines bestimmten Medikamentes die Erfahrung einer schädlichen Reaktion haben. Um diese Behaupting zu testen, beobachtete eine Agentur 2500 Patienten, die dieses Medikament nehmen und beobachtete schädliche Reaktionen in 37 Fällen. Ist das Vorkommen der schädlichen Reaktionen gewöhnlicher als durch das Unternehmen behauptet?

Diese Frage kann als Hypothesentest ausgedrückt werden. Wenn der Populationsanteil der Patienten mit schädlichen Reaktionen als π bezeichnet wird, kann die Hypothese folgendermaßen aufgeschrieben werden...

H0 :   μ  =  128
HA :   μ  ≠  128

Wenn die Null-Hypothese richtig ist, wird die Stichprobenanzahl mit schädlichen Reaktionen eine Binomialverteilung mit den Parametern n = 2500 und π = 0,01 zeigen. Der p-Wert für diesen Test ist deshalb die Summe der Binomialwahrscheinlichkeiten,

p-Wert   =   P(X ≥ 37)   =   P(37)  +  P(38)  +  ... +  P(2499)  +  P(2500)

Viele Möglichkeiten müssen bewertet und summiert werden! Und alle sind nahe Null.

Normale Annäherung

Wir haben bereits gesehen, dass die Normalverteilung als eine Annäherung an die Binomialverteilung genutzt werden kann, wenn n groß ist. Beide, der Stichprobenanteile des Erfolges p und die Anzahl der Erfolg x = np sind ungefähr normal, wenn n groß ist.

Die passendste Normalverteilung kann man benutzen, um eine Annäherung zu jeder Binomial-Restwahrscheinlichkeit zu erhalten. Vor allem kann sie benutzt werden, um eine Annäherung an den p-Wert eines Hypothesentests vorzunehmen.

Das Benutzen einer Annäherung an die Normalverteilung, um den p-Wert zu finden

Eine große Zufallsstichprobe von der Größe n, wird aus einer Population mit der Wahrscheinlichkeit π des Erfolges und x der Erfolge, ausgewählt und beobachtet. Wir werden die Hypothese nochmals testen

H0 :   π  =  π0
HA :   π  <  π0

Der Hauptunterschied von der exakten Methode (das Benutzen der Binomialverteilung) ist das Benutzen einer anderen Teststatistik. Die Teststatistik z erhält man, indem die Anzahl der Erfolge x standardisiert, dass heißt man subtrahiert den Mittelwert der normalen Annäherung und teilt diesen durch seine Standardabweichung.

Schädliche Reaktion auf Medikamente

In diesem Beispiel wurden 37 von 2500 gefunden, die schädliche Reaktionen haben. Der p-Wert, um zu testen, ob die Behauptung des Pharmazie-Unternehmens (dass nur 1% der Patienten schädliche Reaktionen hat), ist die Wahrscheinlichkeit 37 oder mehr. 

Die Stichprobengröße ist groß, wodurch wir eine normale Annäherung verwenden können, um diese Wahrscheinlichkeit zu erhalten, unter der Annahme, dass die zugrundeliegende Populationswahrscheinlichkeit 0,01 ist (die Null-Hypothese).

Stetigkeitskorrektur (fortgeschritten)

Im oberen Teil, konnten wir einen ungefähren p-Wert ermitteln, indem wir eine Test-Statistik benutzten

Mit einer ungefähren Normalverteilung. Da x diskret ist,

P(X ≤ 369)   =  P(X ≤ 369.5)   =   P(X ≤ 369.9)   =   ...

Um diese Restwahrscheinlichkeit zu finden, kann jeder Wert von x zwischen 369 und 370 benutzt worden sein, als man den z-score auswertete. Der p-Wert kann genauer bestimmt werden, indem man 369,5 benutzt. Dies nennt man eine Stetigkeitskorrektur.

Die Stetigkeitskorrektur beinhaltet entweder das Addieren oder Subtrahieren von 0,5 von der beobachteten Zahl x, bevor man den z-score findet.

Sei vorsichtig, wenn es darum geht entweder zu addieren oder zu subtrahieren – das Wahrscheinlichkeits-Statement sollte nicht verändert werden. Zum Beispiel  P(X ≥ 410) = P(X ≥ 409.5), davon sollten 0,5 von x = 410 als eine Stetigkeitskorrektur abgezogen werden, um diese Wahrscheinlichkeit finden zu können, indem man eine Normalannäherung und einen z-score benutzt.

Die Stetigkeitskorrektur ist am Wichtigsten, wenn die beobachtete Zahl nahe an Null oder n ist.
