Ajustes de en (local)

Java

Los diagramas interactivos en CAST son programas de Java. Dado que Java no se puede usar en Android o tabletas de Apple, las versiones interactivas de páginas CAST por lo tanto no están disponibles para usted.

Aunque usted será capaz de ver las páginas de este libro electrónico, sus diagramas interactivos aparecerán en blanco. Ya que CAST se ha estructurado para explicar conceptos estadísticos utilizando sus diagramas interactivos, no será fácil de leer.

Desde este e-book tiene versiones de vídeo o un resumen de sus páginas, usted será capaz de leerlos, pero las versiones interactivas de las páginas no estará disponible.

Desde las versiones interactivas de las páginas de CAST son una parte tan importante del e-libro, le recomendamos que se estudia en un equipo completo (portátil o de escritorio) en lugar de una tableta o un teléfono inteligente.

El material interactivo en el elenco se basa en Java, por lo que es importante instalar y activarlo.

Si un rectángulo verde aparece sin advertencias anteriores, Java está trabajando muy bien. (Se puede tomar un minuto o dos en aparecer.)
Si no aparece, primero asegúrese de que usted tiene la última versión de Java instalada.

Incluso después de que Java se ha instalado, es posible que deba cambiar la configuración del sistema operativo o el navegador con el fin de habilitar completamente (y ver el rectángulo verde arriba sin advertencias).

Habilitar Java 7

Si usted está utilizando la última versión de Java (Java 7), debe estar activado en el Panel de control de Windows. En el escritorio de Windows 8, se puede acceder al panel de control ya sea por clic derecho en la parte inferior izquierda de la pantalla o mantenga presionada la tecla Windows y pulse 'X'. En Windows 7 o versiones anteriores, se accede al panel de control desde el menú Inicio.

Cuando el panel de control está visible, seleccione el Panel de control de Java, a continuación, haga clic en la ficha Seguridad. En la ficha Opciones avanzadas, reducir el nivel de seguridad de Alta a BajaMedia.

Enabling Java 7

If you are using the latest version of Java (Java 7), it must be enabled in the Mac System Preferences. This can usually be found on the Dock.

When the Preferences window is visible, select the Java Preferences, then click the Security tab. In the Advanced tab, reduce the security level from High to LowMedium.

Safari Settings

When running CAST from a locally-installed copy, a few changes must be made to Safari's settings before it will display CAST's interactive diagrams (Java applets).

  • Select Preferences from Safari's File menu.
  • In the Advanced settings, select Show Develop menu in menu bar. Safari will now add an extra menu called Develop to the menu bar.
  • In the Develop menu, the option Disable Local File Restrictions must be set for the interactive diagrams to appear.

Safari on Lion or a more recent version of MacOS X imposes extra "security" restrictions when running a local copy of CAST and these prevent the index and list of data sets from communicating with the main CAST window. If you want to use these features in CAST, we recommend using Firefox.

By default, Safari displays a pop-up warning when you look at any CAST page containing an interactive diagram, asking whether you want to Allow or Block it.

  • Click Allow.
  • Select Preferences from Safari's File menu.
  • In the Security settings, make sure that Allow Java is ticked, then click Manage Website Settings.
  • In the dialog box that appears, there is a blank entry that represents locally installed files. Change its settings to Allow Always.
  • In the dialog box that appears, there is an entry corresponding to the CAST web site. Change its settings to Allow Always.

Firefox Settings

When using Snow Leopard and earlier Mac operating systems, the latest version of Java (Java 7) cannot be installed. With Java 6, Firefox currently shows all interactive diagrams in CAST (Java applets) without the need to change its settings.

When using a locally-installed copy of CAST, Firefox always shows a warning instead of each interactive diagram (Java applet). As a result, every diagram must be clicked to dismiss this warning before it will be displayed.

There is currently no way to tell Firefox to globally enable Java when using a local copy of CAST, so we recommend using a different browser.

By default, Firefox always shows a warning instead of each interactive diagram (Java applet) in CAST. Individual diagrams can be clicked to dismiss these warnings.

These warnings can be dismissed globally for CAST. When a page with an applet is displayed, a red 'Lego block' is shown next to the URL at the top left of the window. Click this to display a message window about the Java plugin. Click the Allow All Plugins dropdown menu and choose Always activate plugins for this site.

Chrome with Snow Leopard and earlier Macs

The latest version of Java (Java 7) cannot be installed on Mac operating systems before Lion, so only Java 6 can be installed. When Chrome first shows a CAST page with an interactive diagram, it asks at the top of the window for permission to run Java.

If you click Always run on this site, Chrome display the interactive diagrams in CAST for the rest of this session. Since this is a locally installed copy of CAST however, Chrome will ask again when CAST is restarted later.

If you click Always run on this site, Chrome display the interactive diagrams in CAST from then on.

Chrome Settings

At present, Chrome does not support the latest version of Java (Java 7). It is possible to install Java 6 and use Chrome, but the best advice at present is to use a different browser. Chrome currently works with Java 6 without any warnings about Java. (See this article about reverting from Java 7 to Java 6.)

However Chrome imposes extra "security" restrictions when running a local copy of CAST and these prevent the index and list of data sets from communicating with the main CAST window. If you want to use these features in CAST, we recommend using Safari or Firefox.
