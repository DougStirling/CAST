Stem and leaf plots

With anything more than a very small data set sampling ordering the data is insufficient to enable to gain much insight into the data.  A stem and leaf plot orders the data and then groups (or stacks) it according to the leading digit (or digits).

If you have not met stem and leaf plots before, we recommend you scroll down and look at the example before reading the detail on how they are constructed.

Note the following for the layout of a stem and leaf plot.

  • The 'axis' is drawn vertically.
  • A value is printed on the axis for each class (stack) giving the most significant digits that are common for all values on that stack.  This is called the stem for that class.
  • The digits representing the values are called the leaf digits and are written in a row to the right of the stems.  Note that each data value can only have 1 leaf digit; other digits are usually 'ignored' (that is, the values are truncated) â€“ see the Human Resources Managers Salaries example.

The layout of a stem and leaf plot makes it particularly easy to read off the values that the leaves represent:

  • Writing the digit displayed for any value (called its leaf digit) after the stack's stem digits gives the most significant digits of the value.

The format is most easily understood with an example.

Note that CAST draws stem and leaf plots with the smallest stem at the bottom. You may have been taught to draw them with the smallest stem at the top. Either ordering is acceptable.
