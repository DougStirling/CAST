CAST on on (local)

Java

The interactive diagrams in CAST are Java programs. Since Java cannot be used on Android or Apple tablets, the interactive versions of CAST pages are therefore not available to you.

Although you will be able to see parts of the pages in this e-book, some of the text and interactive diagrams will not be available. Since CAST has been structured to explain statistical concepts using its interactive diagrams, the e-book will be of very limited use.

Since this e-book has video or summary versions of its pages, you will be able to read them, but the interactive versions of the pages will not be available.

Since the interactive versions of the CAST pages are such an important part of the e-book, we recommend that it is studied on a full computer (laptop or desktop) rather than a tablet or smartphone.

The interactive material in CAST relies on Java, so it is important to install and enable it.

If a green rectangle appears without warnings above, Java is working fine. (Warning: It may take several minutes to download the Java code for CAST when it is first used.)
If the green rectangle does not appear, first ensure that you have the latest version of Java installed.

Even after Java has been installed, you may need to change settings in your operating system or browser in order to fully enable it (and see the green rectangle above without warnings).

Enabling Java 7

The first time a Java applet such as the green rectangle above appears in CAST, you will be asked whether you want to run "castJava" published by Massey University. You should click the checkbox to avoid further messages like this, then click Run in order to use the interactive diagrams in CAST. Your computer should not ask you again.

Internet Explorer Settings

Internet Explorer 8 and earlier do not support web standards for video, so the videos that are present in this e-book cannot be watched. To see the videos, we recommend either using Firefox or Chrome of upgrading to Internet Explorer 9. The original interactive material in CAST still works with IE8 or later.

Click the Tools button then click Internet Options.

  • In the Advanced tab, select the option Allow active content from files on this computer.
  • In the Security tab, turn off the option Enable Protected Mode.

Firefox Settings

By default, Firefox always shows a warning instead of each interactive diagram (Java applet) in CAST.

When a page with an applet is displayed, a 'Lego block' is shown next to the URL at the top left of the window with buttons asking whether applets should be run. Click Allow... then Allow and Remember in the dialog box that appears.

Enabling Java

The first time a Java applet such as the green rectangle above appears in CAST, you will be asked whether you want to run "castJava" published by Massey University. You should click the checkbox to avoid further messages like this, then click Run in order to use the interactive diagrams in CAST. Your computer should not ask you again.

If you do not have an active internet connection, the security settings in Java must be relaxed to run CAST's interactive diagrams. First start the Mac System Preferences — usually found on the Dock.

When the Preferences window is visible, select the Java Preferences, then click the Security tab. In the Advanced tab, reduce the security level from Very High to High.

Safari Settings

When running CAST from a locally-installed copy, a few changes must be made to Safari's settings before it will display CAST's interactive diagrams (Java applets).

  • Select Preferences from Safari's File menu.
  • In the Advanced settings, select Show Develop menu in menu bar. Safari will now add an extra menu called Develop to the menu bar.
  • In the Develop menu, the option Disable Local File Restrictions must be set for the interactive diagrams to appear.

By default, Safari displays a pop-up warning when you look at any CAST page containing an interactive diagram, asking whether you want to Allow or Block it.

  • Click Allow.
  • Select Preferences from Safari's Safari menu.
  • In the Security settings, make sure that Allow Plug-ins is ticked, then click Manage Website Settings.
  • In the dialog box that appears, there is a blank entry that represents locally installed files. From its menu, select Run Always then change its settings to Run in Unsafe Mode.
  • In the dialog box that appears, there is an entry corresponding to the CAST web site. Change its settings to Allow Always.

Firefox Settings

When using Snow Leopard and earlier Mac operating systems, the latest version of Java (Java 7) cannot be installed. With Java 6, Firefox currently shows all interactive diagrams in CAST (Java applets) without the need to change its settings.

By default, Firefox always shows a warning instead of each interactive diagram (Java applet) in CAST.

Chrome Settings

The Chrome browser no longer supports Java so the interactive diagrams in CAST will not work. You are strongly recommended to use Firefox instead.

Browser Settings

Unfortunately recent releases of many popular browsers have removed support for Java, making it impossible to embed CAST's interactive Java material into the pages of this e-book.

This new restriction complicates use of CAST since the interactive material must be displayed using a separate Java application.

Firstly ensure that you have the latest version of Java installed.

Starting the interactive material

You should next start a Java program that is part of the downloaded installation of Java; it can be found in the folder "CAST/core/java" and is called "coreCAST.jar". Double-clicking should start this program. It can be used to display CAST's interactive material and associated usage instructions.

Usage

Firstly select this e-book from the pop-up menu that is displayed. After a short delay, a window should appear with pop-up menus to let you select a chapter, section and page within the e-book. The browser pages in the main CAST e-book will direct you to the appropriate diagrams in this program for study.

Unfortunately the Java program that is needed to run the interactive diagrams in CAST will currently not work when CAST is used from a remote web site. You should download and install CAST for use on a local hard disk or server.
